Write the pull request body for the change that fixes the issue.

Structure:
- **What** — one-sentence summary.
- **Why** — the issue being fixed (link `Closes #<number>`).
- **Changes** — bullet list per file.
- **Verification** — tests run and their result; coverage delta.
- **Trace** — link to the run trace.

Keep it factual and concise. Never include secrets or raw tokens.
